It wasn’t taken down, but the competition itself is over.

The original One Billion Row Challenge (1BRC), created by Gunnar Morling, ran during January 2024. The GitHub repository is still public, but it explicitly states that new submissions are closed and the final leaderboard has been published.  

You can still:
	•	Read the original challenge description.
	•	Download the dataset generator.
	•	Browse all of the submitted implementations.
	•	Study the optimization techniques people used (memory mapping, SIMD, custom parsers, lock-free data structures, cache-friendly hash tables, etc.).
	•	Run the benchmark yourself on your own machine.  

The reason it may feel like it disappeared is that:
	•	The hype peaked in early 2024.
	•	The official competition ended on January 31, 2024.
	•	Discussion shifted into blog posts and language-specific ports (Rust, Go, C, C#, Python, Zig, etc.) rather than new official entries.  

There has even been community interest in reviving it (for example, people discussing doing another edition in 2025), but there hasn’t been a new official 1BRC announced.  

If you’re interested in systems programming or performance engineering, it’s still one of the best practical exercises around. Many people treat it like the performance equivalent of Advent of Code: start with a simple solution that takes minutes, then optimize until you’re processing a 13 GB file in a few seconds.
